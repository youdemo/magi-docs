# 日志系统使用指南

## 快速开始

### 导入
```typescript
import { logger, LogCategory, LogLevel } from '../logging';
```

### 基本使用
```typescript
// 信息日志
logger.info('系统.操作成功', { userId: '123' }, LogCategory.SYSTEM);

// 警告日志
logger.warn('系统.配置缺失', { key: 'apiKey' }, LogCategory.SYSTEM);

// 错误日志
logger.error('系统.操作失败', error, LogCategory.SYSTEM);

// 调试日志
logger.debug('系统.调试信息', { state }, LogCategory.SYSTEM);
```

---

## 日志级别

| 级别 | 用途 | 示例 |
|------|------|------|
| DEBUG | 详细的调试信息 | 变量值、中间状态 |
| INFO | 一般信息 | 操作开始/完成、状态变更 |
| WARN | 警告信息 | 配置缺失、降级处理 |
| ERROR | 错误信息 | 异常、失败 |
| SILENT | 禁用所有日志 | 生产环境静默 |

---

## 日志分类

| 分类 | 用途 | 示例场景 |
|------|------|----------|
| SYSTEM | 系统级日志 | 启动、配置、初始化 |
| CLI | CLI 交互 | 消息发送、响应接收 |
| TASK | 任务管理 | 任务创建、更新、完成 |
| WORKER | Worker 执行 | Worker 启动、任务分配 |
| ORCHESTRATOR | 编排器 | 计划生成、执行协调 |
| SESSION | 会话管理 | 会话创建、切换、保存 |
| RECOVERY | 恢复机制 | 快照、恢复 |
| UI | 用户界面 | 前端交互、状态更新 |

## 常见场景

### 1. 记录操作开始和结束
```typescript
logger.info('任务.执行.开始', { taskId }, LogCategory.TASK);
try {
  // 执行任务
  logger.info('任务.执行.成功', { taskId, result }, LogCategory.TASK);
} catch (error) {
  logger.error('任务.执行.失败', error, LogCategory.TASK);
}
```

### 2. 记录 CLI 交互（基本）
```typescript
// 发送消息
const startTime = Date.now();
logger.logCLIMessage({
  cli: 'claude',
  role: 'worker',
  requestId: 'req-123',
  message: prompt,
  metadata: { taskId: 'task-456' }
});

// 接收响应
const response = await sendMessage(prompt);
logger.logCLIResponse({
  cli: 'claude',
  role: 'worker',
  requestId: 'req-123',
  response: response.content,
  duration: Date.now() - startTime,
  metadata: { taskId: 'task-456' }
});
```

### 2b. 记录 CLI 交互（增强 - 推荐）
```typescript
// 准备消息
const originalPrompt = getUserInput();
const formattedPrompt = formatPromptForCLI(originalPrompt);

// 发送消息（包含完整上下文）
const startTime = Date.now();
logger.logCLIMessage({
  cli: 'claude',
  role: 'worker',
  requestId: 'req-123',
  message: originalPrompt,              // 原始消息
  processedMessage: formattedPrompt,    // 格式处理后的消息
  conversationContext: {
    sessionId: session.id,
    taskId: task.id,
    subTaskId: subtask?.id,
    messageIndex: 2,
    totalMessages: 5
  },
  metadata: { taskId: 'task-456' }
});

// 接收响应
const response = await sendMessage(formattedPrompt);
const parsedResult = parseResponse(response.content);

logger.logCLIResponse({
  cli: 'claude',
  role: 'worker',
  requestId: 'req-123',
  response: response.content,           // 原始响应
  processedResponse: parsedResult,      // 处理后的响应
  duration: Date.now() - startTime,
  conversationContext: {
    sessionId: session.id,
    taskId: task.id,
    subTaskId: subtask?.id,
    messageIndex: 2,
    totalMessages: 5
  },
  metadata: { taskId: 'task-456' }
});
```

### 3. 记录状态变更
```typescript
logger.info('会话.状态变更', {
  from: oldState,
  to: newState,
  reason: 'user action'
}, LogCategory.SESSION);
```

### 4. 记录配置问题
```typescript
if (!config.apiKey) {
  logger.warn('系统.APIKey未配置', {
    configPath: '.multicli/config.json'
  }, LogCategory.SYSTEM);
}
```

### 5. 调试复杂逻辑
```typescript
if (logger.isDebugEnabled(LogCategory.ORCHESTRATOR)) {
  logger.debug('编排器.依赖分析.结果', {
    tasks: tasks.length,
```

## 配置方法

### 方法 1: 环境变量（推荐用于开发）
```bash
# 全局级别
export MULTICLI_LOG_LEVEL=DEBUG

# 分类级别（覆盖全局）
export MULTICLI_LOG_CLI=DEBUG
export MULTICLI_LOG_WORKER=INFO
export MULTICLI_LOG_ORCHESTRATOR=WARN

# 文件输出
export MULTICLI_LOG_FILE=.multicli/logs/app.log
```

### 方法 2: 配置文件
创建 `.multicli/logging.json`:
```json
{
  "level": "INFO",
  "categories": {
    "CLI": "DEBUG",
    "WORKER": "INFO",
    "ORCHESTRATOR": "WARN"
  },
  "file": {
    "enabled": true,
    "path": ".multicli/logs/app.log",
    "maxSize": 10485760,
    "maxFiles": 5
  }
}
```

## 日志文案规范（推荐）

- 使用中文，并保持“模块.动作.阶段”的结构（例如：`UI.任务.执行.开始`）。
- 文案简短可读，详细上下文放在 data 字段。
- 同类事件复用同一文案，便于检索与统计。

### 方法 3: 运行时 API
```typescript
import { logger, LogLevel, LogCategory } from './logging';

// 设置全局级别
logger.setLevel(LogLevel.DEBUG);

// 设置分类级别
logger.setCategoryLevel(LogCategory.CLI, LogLevel.DEBUG);
logger.setCategoryLevel(LogCategory.WORKER, LogLevel.INFO);
```

## 最佳实践

### ✅ 推荐做法

1. **使用合适的级别**
   ```typescript
   // ✅ 正常操作用 INFO
   logger.info('任务创建', { taskId }, LogCategory.TASK);

   // ✅ 调试信息用 DEBUG
   logger.debug('中间状态', { state }, LogCategory.TASK);

   // ✅ 错误用 ERROR
   logger.error('操作失败', error, LogCategory.TASK);
   ```

2. **提供结构化数据**
   ```typescript
   // ✅ 使用对象
   logger.info('用户登录', { userId, ip, timestamp }, LogCategory.SYSTEM);

   // ❌ 避免字符串拼接
   logger.info(`用户 ${userId} 从 ${ip} 登录`);
   ```

3. **选择正确的分类**
   ```typescript
   // ✅ 根据模块选择
   logger.info('任务开始', data, LogCategory.TASK);
   logger.info('Worker 启动', data, LogCategory.WORKER);
   logger.info('CLI 响应', data, LogCategory.CLI);
   ```

4. **使用 CLI 专用方法**
   ```typescript
   // ✅ CLI 交互用专用方法
   logger.logCLIMessage({ cli, role, requestId, message, metadata });
   logger.logCLIResponse({ cli, role, requestId, response, duration, metadata });

   // ❌ 不要用普通日志
   logger.info(`[CLI] 发送消息: ${message}`);
   ```

5. **检查日志级别**
   ```typescript
   // ✅ 昂贵操作前检查
   if (logger.isDebugEnabled(LogCategory.ORCHESTRATOR)) {
     const analysis = performExpensiveAnalysis();
     logger.debug('分析结果', analysis, LogCategory.ORCHESTRATOR);
   }
   ```

### ❌ 避免做法

1. **不要记录敏感信息**
   ```typescript
   // ❌ 不要记录密码、token
   logger.info('登录', { password: '123456' });

   // ✅ 只记录必要信息
   logger.info('登录', { userId, timestamp });
   ```

2. **不要在循环中大量日志**
   ```typescript
   // ❌ 避免
   for (const item of items) {
     logger.debug('处理项目', item);
   }

   // ✅ 汇总记录
   logger.debug('批量处理', { count: items.length, sample: items[0] });
   ```

3. **不要使用 console.*（已禁止）**
   ```typescript
   // ❌ 禁止
   console.log('消息');
   console.error('错误');

   // ✅ 使用 logger
   logger.info('消息');
   logger.error('错误', error);
   ```

## 故障排查

### 问题：日志没有输出

**检查清单**:
1. 检查日志级别配置
   ```bash
   echo $MULTICLI_LOG_LEVEL
   echo $MULTICLI_LOG_CLI
   ```

2. 检查分类是否正确
   ```typescript
   // 确保使用了正确的分类
   logger.info('消息', data, LogCategory.CLI);  // 不是 LogCategory.SYSTEM
   ```

3. 检查是否被 SILENT 级别禁用
   ```bash
   # 临时启用所有日志
   unset MULTICLI_LOG_LEVEL
   ```

### 问题：日志太多

**解决方案**:
1. 提高全局级别
   ```bash
   export MULTICLI_LOG_LEVEL=WARN  # 只显示警告和错误
   ```

2. 禁用特定分类
   ```bash
   export MULTICLI_LOG_CLI=SILENT
   export MULTICLI_LOG_WORKER=SILENT
   ```

3. 只启用需要的分类
   ```bash
   export MULTICLI_LOG_LEVEL=SILENT
   export MULTICLI_LOG_ORCHESTRATOR=DEBUG  # 只看编排器
   ```

## 示例：完整的功能实现

```typescript
import { logger, LogCategory } from '../logging';

export class TaskExecutor {
  async executeTask(taskId: string, prompt: string): Promise<void> {
    // 1. 记录开始
    logger.info('开始执行任务', { taskId, prompt }, LogCategory.TASK);

    try {
      // 2. 调试信息
      if (logger.isDebugEnabled(LogCategory.TASK)) {
        const state = this.getInternalState();
        logger.debug('任务执行前状态', { taskId, state }, LogCategory.TASK);
      }

      // 3. CLI 交互
      const startTime = Date.now();
      logger.logCLIMessage({
        cli: 'claude',
        role: 'worker',
        requestId: `req-${taskId}`,
        message: prompt,
        metadata: { taskId }
      });

      const response = await this.sendToCLI(prompt);

      logger.logCLIResponse({
        cli: 'claude',
        role: 'worker',
        requestId: `req-${taskId}`,
        response: response.content,
        duration: Date.now() - startTime,
        metadata: { taskId }
      });

      // 4. 记录成功
      logger.info('任务执行成功', {
        taskId,
        duration: Date.now() - startTime,
        responseLength: response.content.length
      }, LogCategory.TASK);

    } catch (error) {
      // 5. 记录错误
      logger.error('任务执行失败', error, LogCategory.TASK);

      // 6. 记录恢复尝试
      logger.warn('尝试恢复任务', { taskId, attempt: 1 }, LogCategory.RECOVERY);

      throw error;
    }
  }
}
```
