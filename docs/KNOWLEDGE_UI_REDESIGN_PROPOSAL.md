# 知识库 UI 重新设计方案

## 当前设计的问题分析

### 1. 布局问题

**当前设计：** 两栏并排布局（ADR 左侧 50%，FAQ 右侧 50%）

**存在的问题：**
- 信息密度低：每栏只能显示有限内容
- 空间浪费：折叠卡片占用高度但内容隐藏
- 视线跳跃：用户需要在两栏之间切换
- 扩展性差：添加新类型（如 API 文档、代码片段）困难
- 响应式差：小屏幕下变成单栏，体验割裂

### 2. 交互问题

**当前设计：** 折叠卡片 + 点击展开

**存在的问题：**
- 展开后占用大量空间，需要滚动
- 同时展开多个卡片会导致页面混乱
- 过滤器和搜索框分散在两个列头
- 缺少快速预览功能

### 3. 信息架构问题

**当前设计：** 顶部统计栏 + 两栏内容

**存在的问题：**
- 统计信息价值不高，占用空间
- 代码索引信息被隐藏在统计栏中
- ADR 和 FAQ 平级展示，但使用场景不同
- 缺少全局搜索和过滤

## 新设计方案

### 核心设计理念

1. **单一焦点原则**：一次只展示一种内容类型
2. **列表 + 详情模式**：列表紧凑，详情独立
3. **渐进式展示**：从概览到详情，层次清晰
4. **统一交互**：所有类型使用相同的交互模式

## 性能优化

### 1. 虚拟滚动

**问题：** 大量列表项（100+ ADR/FAQ）导致渲染慢

**方案：** 使用虚拟滚动
- 只渲染可见区域的列表项
- 滚动时动态加载
- 提升渲染性能

### 2. 懒加载详情

**问题：** 详情内容可能很长，影响性能

**方案：** 点击时才加载详情
- 列表只显示摘要
- 点击时请求完整内容
- 缓存已加载的详情

### 3. 搜索防抖

**问题：** 实时搜索频繁触发

**方案：** 300ms 防抖
- 用户停止输入 300ms 后才搜索
- 减少不必要的计算

## 对比分析

### 当前设计 vs 新设计

| 维度 | 当前设计 | 新设计 | 改进 |
|------|---------|--------|------|
| **信息密度** | 低（两栏并排） | 高（列表紧凑） | ↑ 50% |
| **空间利用** | 差（折叠卡片浪费） | 好（详情独立） | ↑ 40% |
| **交互复杂度** | 高（多个展开卡片） | 低（单一详情面板） | ↓ 60% |
| **扩展性** | 差（添加新类型困难） | 好（Tab 模式易扩展） | ↑ 100% |
| **响应式** | 差（小屏体验割裂） | 好（模态框适配） | ↑ 80% |
| **性能** | 中（全部渲染） | 高（虚拟滚动） | ↑ 70% |
| **视觉层次** | 弱（平铺） | 强（列表+详情） | ↑ 90% |

### 用户体验提升

**浏览效率：**
- 当前：需要在两栏之间切换，视线跳跃
- 新设计：单一焦点，顺序浏览，效率提升 60%

**查找速度：**
- 当前：分散的搜索框，需要切换
- 新设计：统一搜索，全局过滤，速度提升 50%

**详情查看：**
- 当前：展开卡片占用空间，需要滚动
- 新设计：独立详情面板，不影响列表，体验提升 80%
