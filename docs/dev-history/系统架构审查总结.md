## 🎯 项目定位

### 核心定位
**MultiCLI** = **基于 Mission-Driven 架构的多模型 AI 协作编排系统**

### 关键特性
1. **多模型协作**: Claude、GPT、Gemini 等多个 AI 模型
2. **智能编排**: 自动任务分解和模型选择
3. **Mission-Driven**: Mission → Assignment → Todo 层次化管理
4. **完整追踪**: 会话、快照、上下文、TODO 文件
5. **知识管理**: ADR、FAQ、代码索引

## 🏗️ 系统架构

```
VSCode Extension
    ↓
UI Layer (Webview)
    ├── 对话界面 (Chat)
    ├── 配置面板 (Config)
    └── 知识库 (Knowledge)
    ↓
Orchestration Layer (编排层)
    ├── IntelligentOrchestrator (智能编排器)
    ├── MissionDrivenEngine (任务驱动引擎)
    ├── MissionOrchestrator (任务分解)
    ├── MissionExecutor (任务执行)
    └── AutonomousWorker (自主 Worker)
    ↓
Adapter Layer (适配器层)
    ├── LLMAdapterFactory (LLM 适配器工厂)
    ├── OrchestratorLLMAdapter (编排者适配器)
    ├── WorkerLLMAdapter (Worker 适配器)
    └── LLMClient (统一客户端)
    ↓
Support Systems (支持系统)
    ├── SessionManager (会话管理)
    ├── SnapshotManager (快照管理)
    ├── ContextManager (上下文管理)

    ├── PlanTodoManager (TODO 管理)
    ├── ConfigManager (配置管理)

```

## ⚠️ 发现的问题

### 1. CLI 模式遗留代码
**位置**: `src/ui/webview-provider.ts`
**问题**:
- `handleCliQuestionAnswer()` 方法使用旧的 `writeInput()` 接口
- LLM 模式不支持此接口
- 已标记 TODO 但未实现

**影响**: 用户无法回答 LLM 的问题

**优先级**: 🔴 高

**建议**:
```typescript
// 需要实现 LLM 模式的问答机制
private async handleLLMQuestionAnswer(
  agent: AgentType,
  questionId: string,
  answer: string,
  adapterRole: 'worker' | 'orchestrator'
): Promise<void> {
  // 将答案作为新消息发送给 LLM
  await this.adapterFactory.sendMessage(
    agent,
    `User's answer: ${answer}`,
    undefined,
    { adapterRole }
  );
}
```

### 2. 部分旧架构代码未清理
**位置**: 多处
**问题**:
- `src/orchestrator/orchestrator-agent.ts` - 旧的编排器
- `src/adapters/cli-adapter-factory.ts` - CLI 适配器
- 部分文件中的 `SubTask` 引用

**影响**: 代码冗余，可能造成混淆

**优先级**: 🟡 中

**建议**:
- 标记为 `@deprecated`
- 添加迁移指南
- 逐步移除

### 3. 测试覆盖不完整
**位置**: `src/test/`
**问题**:
- 缺少 Mission-Driven 架构的集成测试
- 缺少 LLM 适配器的端到端测试
- 部分支持系统缺少单元测试

**影响**: 回归风险

**优先级**: 🟡 中

**建议**:
- 添加 Mission 执行流程的集成测试
- 添加 LLM 适配器的 Mock 测试
- 补充支持系统的单元测试

## 📝 核心数据流

### 1. 任务执行流程
```
用户输入
    ↓
IntelligentOrchestrator (意图识别)
    ↓
MissionDrivenEngine (创建 Mission)
    ↓
MissionOrchestrator (任务分解 → Assignments)
    ↓
MissionExecutor (执行协调)
    ├── 生成 TODO 文件
    ├── 创建快照
    ├── 调用 AutonomousWorker
    └── 更新 TODO 状态
    ↓
AutonomousWorker (执行 Assignment)
    ├── 规划 Todos
    ├── 执行 Todos
    └── 发射事件
    ↓
LLMAdapter (调用 LLM API)
    ↓
LLM 响应
    ↓
结果返回
```

### 2. 数据持久化
```
.multicli/
├── sessions/
│   └── {sessionId}/
│       ├── session.json          # 会话元数据
│       ├── messages.json         # 消息历史
│       ├── snapshots/            # 文件快照
│       │   └── {snapshotId}.snapshot
│       ├── missions/             # Mission TODO 文件
│       │   └── {missionId}.md
│       └── tasks/                # 任务记录
│           └── {taskId}.json
├── knowledge/                    # 项目知识库
│   ├── code-index.json
│   ├── adrs/
│   └── faqs/
└── llm-config.json              # LLM 配置
```
