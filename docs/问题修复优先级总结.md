## P0 问题: 文件冲突检测未启用

### 问题详情

**文档**: [docs/P0问题确认-文件冲突检测未启用.md](./P0问题确认-文件冲突检测未启用.md)

**根本原因**:
1. `TaskDependencyGraph.addTask()` 第4个参数 `targetFiles` 未传递
2. `TaskDependencyGraph.addFileDependencies()` 方法从未调用
3. 虽然代码已实现，但生产环境从未启用

**代码位置**:
- `src/orchestrator/worker-pool.ts:916` - 调用 `addTask()` 时缺少 `targetFiles`
- `src/orchestrator/worker-pool.ts:924` - 应该调用 `addFileDependencies()` 但缺失

**实际风险**:
```typescript
// 危险场景
用户: "同时优化 types.ts 的类型定义和性能"

编排者: 生成两个并行任务
  - task-1: 优化类型定义 (targetFiles: ['src/types.ts'])
  - task-2: 优化性能 (targetFiles: ['src/types.ts'])

实际执行:
  T0: task-1 和 task-2 同时启动 ❌
  T3: task-1 写入 types.ts (版本A)
  T4: task-2 写入 types.ts (版本B) → 覆盖版本A

结果: task-1 的修改全部丢失！
```

**修复方案**:
```typescript
// src/orchestrator/worker-pool.ts

async executeWithDependencyGraph(taskId, subTasks, context) {
  const graph = new TaskDependencyGraph();

  // ✅ 修复1: 传递 targetFiles
  for (const subTask of subTasks) {
    graph.addTask(
      subTask.id,
      subTask.description,
      subTask,
      subTask.targetFiles || []  // 添加第4个参数
    );
  }

  // 添加显式依赖
  for (const subTask of subTasks) {
    if (subTask.dependencies && subTask.dependencies.length > 0) {
      graph.addDependencies(subTask.id, subTask.dependencies);
    }
  }

  // ✅ 修复2: 调用文件依赖检测
  const addedDeps = graph.addFileDependencies('sequential');
  if (addedDeps > 0) {
    console.log(`[WorkerPool] 检测到文件冲突，自动添加 ${addedDeps} 个依赖关系`);
  }

  // 分析依赖图
  const analysis = graph.analyze();
  // ...
}
```

**测试验证**:

## P1 问题: 上下文自动迁移缺失

### 问题详情

**文档**: [docs/P1问题确认-上下文丢失.md](./P1问题确认-上下文丢失.md)

**根本原因**:
- 默认只保留5轮对话（10条消息）
- 超出后使用 `.slice(-maxMessages)` 直接丢弃
- 从未调用 `sessionMemory.addXXX()` 迁移关键信息

**代码位置**:
- `src/context/context-manager.ts:88-100` - `addMessage()` 只丢弃，不迁移

**实际影响**:
```
第1轮: "创建用户管理系统" → 决定使用 React + TypeScript
第2轮: "添加权限管理"
第3轮: "集成数据库"
第4轮: "添加API"
第5轮: "添加邮件验证"
第6轮: "添加密码重置"  ← 第1轮的技术栈决策被丢弃
     → 助手可能使用class组件，与之前不一致 ❌
```

**触发概率**: 80%（普通开发任务5-10轮对话很常见）

**修复方案**:
```typescript
// src/context/context-manager.ts

addMessage(message, applyTruncation = true) {
  // ... 现有代码 ...

  this.immediateContext.push(msg);

  const maxMessages = this.config.immediateContextRounds * 2;
  if (this.immediateContext.length > maxMessages) {
    const systemMessages = this.immediateContext.filter(m => m.role === 'system');
    const otherMessages = this.immediateContext.filter(m => m.role !== 'system');

    // ✅ 新增：提取将被移除的消息
    const toRemove = otherMessages.slice(0, -maxMessages);

    // ✅ 新增：迁移到 Memory
    for (const removedMsg of toRemove) {
      this.migrateToMemory(removedMsg);
    }

    // 清理即时上下文
    this.immediateContext = [
      ...systemMessages,
      ...otherMessages.slice(-maxMessages)
    ];

    console.log(`[ContextManager] 迁移 ${toRemove.length} 条消息到 Memory`);
  }
}

/**
 * ✅ 新增方法: 智能提取关键信息
 */
private migrateToMemory(message: ContextMessage): void {
  if (!this.sessionMemory) return;

  // 用户消息 → 提取需求和决策
  if (message.role === 'user') {
    if (this.containsTechnicalDecision(message.content)) {
      this.sessionMemory.addDecision({
        id: `decision-${Date.now()}`,
        description: this.extractDecisionSummary(message.content),
        reason: '用户需求',
      });
    }
  }

  // 助手消息 → 提取任务和变更
  if (message.role === 'assistant') {
    const taskMatches = message.content.match(/(?:完成|创建|实现)[:：]\s*([^\n]{10,100})/g);
    if (taskMatches) {
      for (const match of taskMatches.slice(0, 3)) {
        this.sessionMemory.addCompletedTask({
          id: `task-${Date.now()}`,
          description: match.substring(0, 100),
        });
      }
    }

    const fileMatches = message.content.match(/(?:创建|修改)\s+([a-zA-Z0-9\-_/\.]+\.[a-z]+)/g);
    if (fileMatches) {
      // ... 提取文件变更
    }
  }
}
```

- ✅ 长会话不再丢失早期信息
- ✅ 技术决策自动保存到 Memory

// src/cli/adapter-factory.ts

private emitOrchestratorMessage(type: CLIType, message: string): void {
  const summary = this.summarizeOrchestratorMessage(message);

  // ✅ 使用 Normalizer 替代 output 事件
  const normalizer = this.getOrCreateNormalizer(type, 'orchestrator');
  const traceId = this.generateTraceId();
  const messageId = normalizer.startStream(traceId, 'orchestrator');
  normalizer.processChunk(messageId, summary);
  normalizer.endStream(messageId);

## P2 问题: 压缩器未实现

### 问题详情

**实际状况**:
- `ContextManager.needsCompression()` 存在 ✅
- 但无 `compress()` 实现 ❌

**影响**:
- Memory 可能无限增长
- 长期会话后，上下文获取变慢

**修复方案**:
```typescript
async compressMemory(): Promise<void> {
  if (!this.sessionMemory) return;

  const content = this.sessionMemory.getContent();
  if (content.tokenEstimate < this.config.compression.tokenLimit) {
    return;
  }

  // 使用 Claude 压缩任务历史
  const prompt = `
请压缩以下任务历史为简洁摘要（不超过原文30%）：
${content.completedTasks.map(t => `- ${t.description}`).join('\n')}
`;

  const summary = await this.cliAdapter.sendMessage(prompt);
  this.sessionMemory.replaceCompletedTasks([{
    id: 'summary',
    description: summary,
  }]);
}
```

**优先级**: P2 - 后续优化（长期任务才需要）
