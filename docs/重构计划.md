# MultiCLI 编排重构计划

本文档定义了目标架构、重构里程碑和任务列表，用于迁移到长生命周期 CLI 会话模型，包含统一协议、队列、健康恢复和角色隔离。

## 目标

- 每个模型（claude/codex/gemini）使用长生命周期 CLI 会话，保持上下文连续性
- 每个 CLI 进程使用单写入者队列，避免并发冲突
- 统一消息协议（开始/结束标记 + JSON 元数据）
- 可靠的取消/中断机制，支持软停止和硬重启
- 健康监控和自动恢复，可重放最后的上下文
- 编排者和工作者角色隔离（独立会话/进程）
- Token 高效的内存管理：自动摘要 + 内存注入
- 确定性的 UI 事件流，包含完整对话时间线

## 目标架构

### 1) 会话管理器（每个 CLI）
- 每个 CLI 类型一个长生命周期进程（每个会话范围）
- 队列输入，使用显式结束标记消费输出
- 崩溃时使用上下文快照 + 内存注入恢复

### 2) 统一协议
- 每个 CLI 调用的结构化信封：
  - request_id, task_id, subtask_id, role, intent, timestamp
  - stop_marker（唯一）
  - meta: tool_policy, file_lock, memory_refs
- 输出解析：
  - 流式传输到 UI
  - 通过 stop_marker 检测完成
  - 捕获工具调用、错误、Token 使用情况

### 3) 调度器 + 队列
- 每个 CLI 的 FIFO 队列，支持优先级和文件锁
- 全局规划器拆分任务，然后分发到 CLI 队列
- 安全时并行执行；文件级锁时串行执行

### 4) 健康 + 恢复
- 每个 CLI 进程的心跳检测
- 空闲超时检测 + 自动重启
- 重启时：恢复最后的内存摘要 + 关键上下文

### 5) 审查/验证管道
- 自检/互检可选且由策略驱动
- 集成审查作为独立阶段，仅在需要时执行
- 简单任务避免审查（快速路径）

### 6) UI 事件流
- 编排者 <-> 工作者消息的完整时间线
- 工具调用同时显示请求和结果
- 特殊面板（md/code/tool）的一致渲染

## 重构里程碑

### 里程碑 A：协议 + 会话核心
- 引入长生命周期 CLI 会话抽象
- 实现统一信封 + stop_marker 处理
- 提供会话生命周期 API：start, send, interrupt, reset

### 里程碑 B：队列 + 调度器
- 每个 CLI 的队列，支持文件锁和优先级
- 通过调度器分发；移除每次调用的进程生成

### 里程碑 C：编排者集成
- 编排者使用会话 API 和队列
- 集中应用审查策略 + 集成策略
- 集成上下文压缩和内存注入

### 里程碑 D：UI + 可观测性
- 为每个步骤发出结构化事件
- 修复工具面板排序和状态更新
- 添加统计：每个 CLI 的 Token、延迟、重试次数

## 验收标准

- 每个 CLI 的长生命周期会话在多个提示中保持稳定
- 编排者任务完成时无进程抖动
- 工具面板按顺序显示请求 + 结果
- 取消/恢复正常工作；无孤立的 CLI 进程
- Token 和请求统计实时更新

## 执行说明

- 重构将移除"每条消息生成进程"的用法
- 现有适配器将被包装或替换
- 测试应包括：简单询问、多任务、取消、恢复
