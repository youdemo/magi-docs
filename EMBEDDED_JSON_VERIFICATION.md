# 裸露 JSON 修复验证报告

## 问题回顾

用户报告看到了截图中的问题：Orchestrator 响应中显示了一个 JSON 代码块，包含 "constraints" 约束条件。

## 修复内容

### 新增功能

**文件**: `src/utils/content-parser.ts`

1. **`extractEmbeddedJson()`** - 提取内容中的裸露 JSON 对象
2. **`tryExtractJsonAt()`** - 使用括号匹配算法提取 JSON
3. **修改 `parseContentToBlocks()`** - 在解析前移除裸露的 JSON

### 工作原理

```
原始内容:
  "文字说明\n\n{JSON对象}\n\n后续文字"
    ↓
extractEmbeddedJson() 提取 JSON 位置
    ↓
从后往前移除 JSON
    ↓
最终内容:
  "文字说明\n\n后续文字"
```

## 与之前修复的关系

这是 JSON 处理的第三层防护：

1. **第一层**（前端）: 移除前端 JSON 检测 - 不再将 text 块中的 JSON 渲染为代码块
2. **第二层**（后端）: 只检测纯 JSON - 混合内容不会被误判为 JSON
3. **第三层**（后端-新增）: 移除裸露 JSON - 直接从内容中移除未包裹的 JSON 对象
