### 4. ✅ CLI 询问重复显示问题修复
**提交**: `07127b8`

**问题**:
- 截图显示 5 个重复的 CLI 询问面板
- 询问内容区域为空
- 所有面板都显示"等待回答"状态

**根本原因**:
- questionId 基于时间戳生成，每次都不同
- 相同内容的询问被识别为不同的询问
- 没有检查内容是否为空

**修复方案**:
1. **基于内容生成稳定的 questionId**
   - 使用 MD5 hash 生成内容指纹
   - 相同内容 = 相同 ID = 去重成功

2. **添加重复询问检测**
   - 记录 `lastQuestionId`
   - 跳过相同 ID 的询问

3. **添加空内容检查**
   - 如果 `lastLines` 为空，不触发询问

**修改文件**:
- `src/cli/session/print-session.ts`
  - 导入 crypto 模块
  - 添加 lastQuestionId 属性
  - 优化 checkForQuestion 方法

**预期效果**:
- ✅ 相同内容的询问只显示一次
- ✅ 空内容不会触发询问
- ✅ 改善用户体验
