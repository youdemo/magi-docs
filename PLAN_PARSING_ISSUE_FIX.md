## 根本原因分析

### 1. Claude 返回了解释性文本而不是 JSON

**Claude 的回复**:
```
我现在处于计划模式，需要先探索项目现状以制定详细的实施计划。
让我启动探索代理来了解当前的代码库结构。
```

**期望的回复**:
```json
{
  "analysis": "需要探索项目结构",
  "isSimpleTask": false,
  "needsWorker": true,
  ...
}
```

### 2. 为什么 Claude 会返回解释性文本？

可能的原因：
1. **提示词不够强制**：虽然说了"请仅输出严格 JSON"，但没有明确禁止解释
2. **"工具限制"引起混淆**：提示词中说"禁止调用任何工具"，Claude 可能认为需要先解释为什么不能使用工具
3. **"计划模式"触发特殊行为**：Claude 可能将"计划模式"理解为需要先解释计划思路
4. **缺少明确的格式约束**：没有明确说"第一个字符必须是 {"

### 3. 修复尝试为什么失败？

系统发送了修复提示：
```
你刚才的输出不是合法 JSON。
请将以下内容修复为**严格 JSON**，仅输出 JSON，不要代码块或解释文本。
```

但 Claude 可能：
- 仍然返回了解释性文本
- 或者返回了带代码块标记的 JSON（```json）
- 导致第二次解析也失败

---

## 解决方案

### 已实施的修复

#### 1. 优化提示词 - 添加强制约束

**修改文件**: `src/orchestrator/prompts/orchestrator-prompts.ts`

**修改内容**:
```typescript
## 强制规则
1. 若任务涉及前后端协作（或同时需要 Codex 与 Gemini），必须包含 **Claude 架构/契约任务**，并让其他子任务依赖该任务。
2. 架构任务需明确：目录结构、接口契约、前后端对接约束。
3. **禁止返回解释性文本**。不要说"我现在..."、"让我..."、"首先需要..."等。

## 输出格式 - 严格要求
**关键：你的回复必须是纯 JSON，第一个字符是 {，最后一个字符是 }**
**不要输出任何解释、前缀、后缀或代码块标记（如 ```json）**
**直接输出以下 JSON 对象：**
```

**改进点**:
1. ✅ 明确禁止解释性文本（"我现在..."、"让我..."）
2. ✅ 强制第一个字符是 {，最后一个字符是 }
3. ✅ 禁止代码块标记（```json）
4. ✅ 移除可能引起混淆的"工具限制"标题
5. ✅ 合并到"强制规则"中，更加清晰

## 其他可能的改进（未来）

### 1. 增强 JSON 提取逻辑

当前代码：
```typescript
private parseExecutionPlan(rawContent: string): ExecutionPlan | null {
  const match = rawContent.match(/\{[\s\S]*\}/);
  if (!match) return null;

```

**改进建议**:

- 移除代码块标记（```json）
- 移除前后的解释性文本
- 使用更智能的 JSON 边界检测

### 2. 改进修复提示词

当前修复提示可能不够强制，可以：
- 添加示例 JSON
- 明确说明错误原因
- 提供正确的格式模板

### 3. 添加重试机制

- 第一次失败：尝试修复
- 第二次失败：使用规则引擎生成计划（fallback）
- 记录失败原因，用于后续优化

### 4. 监控和告警

- 记录 JSON 解析失败率
- 分析失败的原因分布
- 定期优化提示词
